# SnapStack 상세 기능명세서 (Functional Specification)

> 알캡쳐(ALCapture)류 Windows 화면 캡쳐·주석·클립보드 도구. 본 문서는 [개발계획서](DEVELOPMENT_PLAN.md)를 구현 가능한 수준으로 구체화한 기능 명세다.

| 항목 | 내용 |
|------|------|
| 프로젝트 | **SnapStack** |
| 플랫폼 | Windows 10 / 11 (x64) 전용 |
| 스택 | C# 12, .NET 8 (LTS), WPF, MVVM |
| 문서 버전 | v0.1 (2026-06-16) |
| 상태 | 명세 확정 — 구현 착수 단계 |
| 작성 | 팀 오케스트라(capture·editor·clipboard·system) 분담 작성 → 지휘자 통합 |

## 필수 4대 기능 추적표 (검수 기준)

| # | 필수 기능 | 명세 위치 | 우선순위 |
|---|-----------|-----------|----------|
| 1 | 캡쳐 히스토리(**순서대로** 표시) | §4.1 HIS-01~06 | Must |
| 2 | 페인팅/주석 편집 | §3 EDT-01~11 | Must |
| 3 | 클립보드 복사 | §4.2 CLP-01 | Must |
| 4 | 타 프로그램 붙여넣기 | §4.3 PST-01 | Must |

기능ID 접두어: 캡쳐 `CAP`, 편집 `EDT`, 히스토리 `HIS`, 클립보드 `CLP`, 붙여넣기 `PST`, 저장 `SAV`, 시스템 `SYS`, 비기능 `NFR`. 우선순위는 MoSCoW(Must/Should/Could).

## 1. 개요

### 1.1 목적·범위
SnapStack은 "**찍고(Snap) → 쌓고(Stack) → 그리고 → 붙여넣는다**"는 단일 워크플로를 매끄럽게 잇는 개인용 캡쳐 도구다. 본 명세는 v1.0(MVP) 범위를 다룬다. 스크롤 캡쳐·화면 녹화·OCR·클라우드는 §개발계획서 §2.2대로 v1 제외.

### 1.2 핵심 데이터 계약 (전 영역 공통 — capture·editor·clipboard 확정)
모든 영역이 이 계약 위에서 동작한다(§4.0의 단일 출처).

| 계약 | 요지 |
|------|------|
| **적재**(capture→history) | 캡쳐는 픽셀+메타(`Original` Frozen BitmapSource, `PixelSize`, `CapturedAt`, 캡쳐모드)만 넘긴다. **`Sequence`는 HistoryService가 Add 순서로 부여**(표시·정렬 SSOT), `Id`(Guid)는 캡쳐가 부여. |
| **평탄화**(editor→clipboard/file) | 복사·저장은 **합성된 단일 비트맵**을 입력으로 받는다. `EditorViewModel.FlattenToBitmap()`이 모든 레이어+자르기를 합성해 **Pbgra32·원본 PixelSize 1:1·96dpi·frozen** `BitmapSource` 반환. 출력 서비스는 **인코딩만**. |
| **CaptureItem** | `{ Id, Sequence, CapturedAt, Original(Frozen·Pbgra32), PixelSize, IsEdited, EditedComposite(편집시 채움·Frozen·nullable) }` + 히스토리 내부 `Thumbnail`·`TempFilePath`. **모든 BitmapSource는 Frozen.** |
| **EffectiveImage** | `EffectiveImage = (IsEdited && EditedComposite!=null) ? EditedComposite : Original`. 복사·저장·미리보기·썸네일이 **전부 이 규칙** 참조. |

### 1.3 화면 구성(개관)
- **메인 윈도우**: 상단 캡쳐 도구바 + 하단 **히스토리 썸네일 스트립**(가로 스크롤) + 중앙 미리보기.
- **캡쳐 오버레이**: 전체 가상화면을 덮는 투명 창(영역 선택).
- **편집기 윈도우**: 좌측 도구바 + 중앙 캔버스 + 상단 옵션바.
- **시스템 트레이**: 상주 아이콘 + 우클릭 메뉴.
- **설정 윈도우**: 탭(일반/단축키/저장/캡쳐/정보).

## 2. 캡쳐 기능 — `CAP`

> 화면의 일부 또는 전체를 비트맵으로 취득해 **Frozen Original**로 히스토리에 적재(§1.2 적재 계약)한다. 캡쳐는 항상 **단일 진행 가드**(동시 캡쳐 금지)를 지킨다.

### 2.1 캡쳐 모드

#### CAP-01 · 영역 지정 캡쳐 (Must) ★기본
- **설명**: 드래그로 사각 영역을 선택해 캡쳐. 가장 많이 쓰는 기본 모드.
- **트리거**: 전역 `Ctrl+Shift+A` / 트레이·메인 버튼.
- **동작 흐름**: 트리거 → 전체 가상화면 스냅샷을 배경으로 한 오버레이(CAP-07) 표시 → 드래그로 영역 지정 → `Enter`/더블클릭 확정 → 해당 영역 BitBlt → Frozen Original 산출 → 히스토리 Add → (설정) 편집기 자동 열기 또는 미리보기.
- **입력**: 마우스 드래그(시작점→끝점). **출력**: 선택 영역 크기의 `BitmapSource`.
- **예외/엣지**: 0×0/Esc 취소 → 미적재. 모니터 경계 가로지르는 영역 허용(가상좌표 기준).
- **수용기준**: 임의 영역 드래그·확정 시 픽셀 정확히 일치하는 캡쳐가 히스토리에 1건 추가된다.
- **구현 힌트**: `Graphics.CopyFromScreen(srcX, srcY, …)` (가상화면 좌표), 결과 `Bitmap`→`BitmapSource` 변환 후 `Freeze()`.

#### CAP-02 · 활성 창 / 단위 영역 캡쳐 (Must)
- **설명**: 마우스 아래의 창/컨트롤을 자동 인식해 외곽을 하이라이트, 클릭 한 번에 그 영역 캡쳐.
- **트리거**: 전역 `Ctrl+Shift+W`, 또는 오버레이에서 드래그 없이 호버 → 클릭.
- **동작 흐름**: 오버레이 상태에서 마우스 이동 → `WindowFromPoint`로 창 핸들 탐지 → `GetWindowRect`(DWM 확장 프레임 고려)로 외곽 하이라이트 → 클릭 시 그 사각형 캡쳐.
- **예외/엣지**: 멀티 DPI 창의 프레임 오차 → DWM `DwmGetWindowAttribute(EXTENDED_FRAME_BOUNDS)` 사용. 최소화 창 제외.
- **수용기준**: 임의 창 위에서 외곽이 정확히 점등되고, 클릭 시 그 창만 캡쳐된다.

#### CAP-03 · 전체 화면 캡쳐 (Must)
- **설명**: 현재 모니터 전체 또는 전체 가상화면(모든 모니터)을 캡쳐.
- **트리거**: 현재 모니터 `Ctrl+Shift+F`, 전체 가상화면 `Ctrl+Shift+G`.
- **동작 흐름**: 트리거 → 마우스가 있는 모니터(또는 가상화면 전체) 경계 산출 → 즉시 BitBlt → 적재. (오버레이 불필요)
- **입력**: 없음(키만). **출력**: 모니터/가상화면 크기 비트맵.
- **예외/엣지**: 혼합 DPI에서 각 모니터 물리 해상도 합산(가상화면 좌표계).
- **수용기준**: 다중 모니터에서 `Ctrl+Shift+F`는 커서가 있는 모니터만, `Ctrl+Shift+G`는 전체를 캡쳐.

#### CAP-04 · 직전 영역 재캡쳐 (Should)
- **설명**: 마지막으로 캡쳐한 영역 좌표를 그대로 다시 캡쳐(반복 캡쳐 가속).
- **트리거**: 전역 `Ctrl+Shift+R`.
- **동작 흐름**: 마지막 캡쳐의 사각형(가상좌표) 기억 → 트리거 시 오버레이 없이 즉시 그 영역 BitBlt → 적재.
- **예외/엣지**: 직전 영역 없음(최초 실행) → 토스트 안내, 영역 캡쳐로 폴백.
- **수용기준**: 영역 캡쳐 후 `Ctrl+Shift+R`로 동일 좌표 캡쳐가 즉시 추가된다.

#### CAP-05 · 지연(타이머) 캡쳐 (Should)
- **설명**: N초 카운트다운 후 캡쳐(메뉴 펼침·툴팁 등 사라지는 UI 캡쳐용).
- **트리거**: 전역 `Ctrl+Shift+D`, 지연 시간은 설정(기본 3초).
- **동작 흐름**: 트리거 → 트레이/오버레이에 카운트다운 표시 → 만료 시 지정 모드(기본 전체화면 또는 직전영역)로 캡쳐.
- **예외/엣지**: 카운트다운 중 다른 캡쳐가 끼면 **완료 순서**로 Sequence 부여(§HIS-01). 취소 키 `Esc`.
- **수용기준**: 3초 후 자동 캡쳐가 1건 적재된다.

### 2.2 횡단 요구

#### CAP-06 · 다중 모니터 / Per-Monitor DPI (Must) ★리스크
- **설명**: 서로 다른 배율(100/150/175%)·해상도의 다중 모니터에서 캡쳐 좌표가 정확해야 한다.
- **요구**: ① `app.manifest`에 **PerMonitorV2** 선언. ② 모든 좌표를 **가상화면 물리 픽셀** 단일 좌표계로 환산(WPF DIP↔물리 픽셀 변환을 한 헬퍼에 집중). ③ 오버레이도 가상화면 전체를 물리 픽셀로 덮는다.
- **예외/엣지**: 음수 좌표 모니터(주 모니터 왼쪽/위), 배율 경계를 가로지르는 영역.
- **수용기준**: 100%+150% 혼합 환경에서 보조 모니터의 특정 영역을 캡쳐해도 1픽셀 어긋남 없이 일치.
- **구현 힌트**: `GetSystemMetrics(SM_XVIRTUALSCREEN/SM_CXVIRTUALSCREEN…)`, `PresentationSource`/`VisualTreeHelper.GetDpi`.

#### CAP-07 · 영역선택 오버레이 UX (Must)
- **설명**: 영역/창 캡쳐 시 뜨는 전체화면 선택 UI.
- **요구**: ① 캡쳐 시점 화면 스냅샷을 배경으로 깔고 그 위에 **반투명 어둠 마스크**, 선택 영역만 원본 밝기로. ② 드래그 중 **실시간 치수(px)·좌표** 표시. ③ 커서 주변 **확대경(loupe)** + 픽셀 색상(Should). ④ `Enter`/더블클릭 확정, `Esc` 취소, 방향키 1px 미세조정(Should). ⑤ 오버레이 내 즉시 동작: `Ctrl+Enter` 즉시 복사(§CLP-01), `Shift+Enter` 즉시 저장(§SAV-01).
- **수용기준**: 마스크·치수가 실시간 표시되고, 확정 시 정확한 영역이 캡쳐된다.
- **구현 힌트**: `Window`(`WindowStyle=None`, `AllowsTransparency=True`, `Topmost=True`, 가상화면 전체 덮기), 배경 스냅샷 위 `Canvas` 드로잉.

### 2.3 캡쳐 단축키 요약
| 동작 | 기본 단축키(전역) | 비고 |
|------|------|------|
| 영역 지정 캡쳐 | `Ctrl+Shift+A` | 기본 모드 |
| 활성 창 캡쳐 | `Ctrl+Shift+W` | 호버 하이라이트 |
| 전체 화면(현재 모니터) | `Ctrl+Shift+F` | |
| 전체 가상화면 | `Ctrl+Shift+G` | 모든 모니터 |
| 직전 영역 재캡쳐 | `Ctrl+Shift+R` | |
| 지연 캡쳐 | `Ctrl+Shift+D` | 기본 3초 |

> 전역키 체계는 **Ctrl+Shift+\* 통일안(B안)** 채택 — OS `PrtScn` 충돌 회피. `PrtScn`은 설정에서 영역 캡쳐 보조키로 등록 가능. 전부 §5 설정에서 재지정.

## 3. 이미지 편집/페인팅 — `EDT` (필수 기능 #2)

> 선택한 캡쳐 위에 주석을 그리는 편집기. 자유곡선은 `InkCanvas`, 도형/텍스트/번호는 `Canvas`+`Adorner` 객체로 관리(개별 이동·삭제·Undo), 모자이크는 `WriteableBitmap` 픽셀 처리. **편집 결과는 §1.2 평탄화 계약대로 `FlattenToBitmap()`으로 합성**해 EditedComposite로 저장한다.

### 3.1 드로잉 도구

#### EDT-01 · 펜 / 브러시 (Must)
- **설명**: 자유곡선 그리기. 색·굵기 조절.
- **옵션**: 색상, 굵기(1~20px), (Should)부드러움. **단축키**: `P`.
- **수용기준**: 드래그 경로가 매끄러운 선으로 그려지고 Undo로 한 획씩 취소된다.
- **구현 힌트**: `InkCanvas` `DrawingAttributes`(Color/Width).

#### EDT-02 · 형광펜 (Must)
- **설명**: 반투명 강조선(겹쳐도 누적되지 않는 곱하기 느낌).
- **옵션**: 색상(기본 노랑), 굵기, 투명도(기본 ~40%). **단축키**: `H`.
- **수용기준**: 텍스트 위에 반투명으로 덧칠되어 아래 내용이 비친다.
- **구현 힌트**: `InkCanvas` + `IsHighlighter=true`.

#### EDT-03 · 도형: 사각형 / 타원 (Must)
- **설명**: 외곽선/채움 사각형·타원.
- **옵션**: 선 색·굵기, 채움 on/off·채움색, `Shift`=정사각/정원. **단축키**: `R`(사각), `O`(타원).
- **수용기준**: 드래그로 그린 도형을 핸들로 이동/리사이즈하고 Undo된다.
- **구현 힌트**: `Canvas` 위 `Rectangle`/`Ellipse` + 선택/리사이즈 `Adorner`.

#### EDT-04 · 직선 / 화살표 (Must)
- **설명**: 직선과 화살표(머리 표시). 캡쳐 주석의 핵심.
- **옵션**: 색·굵기, 화살표 머리 크기, `Shift`=15°스냅. **단축키**: `L`(직선), `A`(화살표).
- **수용기준**: 시작→끝 드래그로 화살표가 그려지고 끝점에 머리가 표시된다.

#### EDT-05 · 텍스트 (Must)
- **설명**: 클릭 위치에 텍스트 상자 입력.
- **옵션**: 폰트·크기·색·굵기/이탤릭, 배경/외곽(Should). **단축키**: `T`.
- **동작/엣지**: 입력 중에는 단일키 단축키 비활성(타이핑 우선), `Esc`/외부 클릭으로 확정. 빈 텍스트는 폐기.
- **수용기준**: 클릭→타이핑→확정으로 텍스트가 배치되고 재선택해 수정/이동된다.

#### EDT-06 · 번호 스탬프 (Should)
- **설명**: 클릭마다 ①②③ 자동 증가하는 번호 뱃지(절차 설명용).
- **옵션**: 시작 번호, 색·크기, 모양(원/사각). **단축키**: `N`.
- **수용기준**: 연속 클릭 시 1,2,3…이 배치되고 삭제 시 후속 번호 재정렬(Should).

#### EDT-07 · 모자이크 / 블러 (Must)
- **설명**: 민감 영역 가리기(픽셀화 또는 가우시안 블러).
- **옵션**: 모자이크 셀 크기 / 블러 강도, 브러시·영역 모드. **단축키**: `M`.
- **수용기준**: 지정 영역이 식별 불가하게 처리되고 Undo로 복원된다.
- **구현 힌트**: 해당 영역 `WriteableBitmap` 다운샘플 후 업스케일(모자이크) / 컨볼루션(블러). `[`/`]`로 강도 조절.

#### EDT-08 · 자르기 (Crop) (Must)
- **설명**: 직사각형 영역으로 이미지 잘라내기(v1 직사각형만).
- **옵션**: 핸들 드래그 조정, 확정/취소. **단축키**: `C`.
- **수용기준**: 자른 영역만 남고 캔버스·이후 합성 크기가 갱신된다.

### 3.2 공통 편집 기능

#### EDT-09 · 색상 / 굵기 / 투명도 (Must)
- 옵션바에서 활성 도구의 색·굵기·투명도·채움 조절. 최근 색 팔레트(Should). `[`/`]`=활성 도구 크기·강도.

#### EDT-10 · 실행취소 / 재실행 (Must)
- **설명**: 모든 드로잉 동작을 Command 스택으로 관리.
- **단축키**: `Ctrl+Z`(undo) / `Ctrl+Y`·`Ctrl+Shift+Z`(redo).
- **수용기준**: 10단계 이상 정확히 되돌리고 다시 적용된다.

#### EDT-11 · 편집기 레이아웃 / 동작 (Must)
- **구성**: 좌측 도구바(아이콘) · 중앙 캔버스(원본 1:1) · 상단 옵션바 · 하단 상태바(좌표/크기).
- **동작/단축키**: 복사 `Ctrl+C`(§CLP-01), 저장 `Ctrl+S`/`Ctrl+Shift+S`(§SAV), 복제 `Ctrl+D`, 창 맞춤 `Ctrl+9`, 닫기 `Ctrl+W`(미저장 시 확인), 선택 삭제 `Delete`/`Backspace`. **`Esc`는 창을 닫지 않고** 선택 해제/도구 취소.
- **저장 시점**: "저장/복사" 시 `FlattenToBitmap()`로 합성 → `EditedComposite`(Frozen) 채우고 `IsEdited=true` → HistoryService에 썸네일 재생성 통지(§HIS-03).
- **v1 한계**: 닫으면 드로잉 객체(레이어) 폐기 → 개별 주석 재편집 불가, 합성본만 영속(§HIS-05).

## 4. 히스토리 / 클립보드 / 붙여넣기 / 저장 — `HIS`·`CLP`·`PST`·`SAV`

> 본 장은 SnapStack 필수 4대 기능 중 **3개**(#1 캡쳐 히스토리, #3 클립보드 복사, #4 외부 앱 붙여넣기)와 파일 저장을 다룬다. "찍고 → **쌓고** → 그리고 → **붙여넣는다**"의 뒷 절반이며 제품의 차별점이자 검수 기준이다.

### 4.0 데이터 계약 (capture·editor·clipboard 3자 확정)

| 계약 | 내용 |
|------|------|
| **적재 계약** (capture→history) | 캡쳐(영역/창/전체/직전/지연)는 성공 시 픽셀데이터+메타만 넘긴다: `Original`(BitmapSource), `PixelSize`, `CapturedAt`, 캡쳐모드. **Sequence·Id 중 Sequence는 HistoryService가 Add 호출 순서로 부여**(표시·정렬 SSOT), Id는 캡쳐가 Guid 부여. **입력 BitmapSource는 Frozen 전제**(캡쳐가 백그라운드에서 `Freeze()`). 마샬링·썸네일 생성·영속화는 히스토리 책임. |
| **평탄화 계약** (editor→clipboard/file) | "복사/저장"은 **합성된 단일 비트맵**을 입력으로 받는다. EditorViewModel이 `FlattenToBitmap()`으로 모든 레이어(Ink·도형·텍스트·번호·모자이크)+자르기를 **on-demand 합성**해 단일 `BitmapSource`(**Pbgra32, 원본 PixelSize 1:1, 96dpi, frozen**)를 반환(렌더 1회 캐시·편집변경 시 dirty 무효화). 출력 서비스는 **인코딩만** 한다. |
| **CaptureItem 스키마** | `{ Id(캡쳐), Sequence(히스토리), CapturedAt(캡쳐), Original(캡쳐·Frozen·Pbgra32), PixelSize(캡쳐), IsEdited, EditedComposite(편집시 editor가 채움·Frozen·nullable) }` + 히스토리 내부 `Thumbnail`·`TempFilePath`. **모든 BitmapSource는 Frozen.** |
| **EffectiveImage 단일 규칙** | `EffectiveImage = (IsEdited && EditedComposite != null) ? EditedComposite : Original`. **복사·저장·미리보기·썸네일 재생성 전부 이 규칙 참조**(IsEdited 분기 중복 제거). EditedComposite null이면 Original 자동 폴백. |
| **서비스 API** | `bool ClipboardService.CopyImage(BitmapSource)` / `bool FileExportService.Save(BitmapSource, ImageFormat, …)`. 성공/실패 bool 반환, **토스트는 각 호출 측이 표시**. |

### 4.1 캡쳐 히스토리 (필수 기능 #1) — `HIS`

> 캡쳐한 이미지를 **캡쳐한 순서대로** 썸네일 스트립에 누적하고, 언제든 다시 꺼내 쓰는 워크플로. SnapStack의 1급 차별 기능.

#### HIS-01 · 순서대로 누적 표시 (Must)
- **설명**: 새 캡쳐가 발생하면 히스토리 컬렉션에 항목을 추가하고 썸네일 스트립(가로 스크롤)에 노출한다. 표시 순서는 **Sequence(단조 증가)** 를 따른다.
- **동작 흐름**: 캡쳐 완료(Frozen Original 산출) → `HistoryService.Add(original, meta)` → `Sequence = ++_counter` 부여 → 썸네일 생성 → `ObservableCollection<CaptureItemVM>`에 삽입 → 스트립이 **최신 항목으로 자동 스크롤**.
- **입력**: Frozen `BitmapSource Original`, `Size PixelSize`, `DateTime CapturedAt`, 캡쳐모드. **출력**: 스트립에 추가된 썸네일 1개.
- **표시 방향**: 기본 **오래된 것 → 새 것을 오른쪽으로 누적**(알캡쳐 동일), 우측 자동 스크롤. 정렬 방향 토글(Could).
- **순번/스레드 계약**: Sequence는 **`Add()` 호출 순서(완료 순서)** 로 부여. 캡쳐는 **직렬화(단일 진행) 가드**(영역/창은 모달 오버레이라 구조상 단일; 전체/직전/지연은 '진행 중 새 트리거 무시 또는 큐잉'). 캡쳐 완료 콜백이 백그라운드일 수 있으므로 **`HistoryService.Add` 내부에서 Dispatcher 마샬링**으로 컬렉션 삽입. 입력이 Frozen이라 교차 스레드 안전.
- **예외/엣지**: ① 0×0/취소 → 미적재. ② 지연 캡쳐 카운트다운 중 다른 캡쳐가 끼면 완료 순서로 Sequence 일관(트리거 시각과 어긋나도 완료 순서 우선). ③ 대형 캡쳐 → 썸네일만 즉시, 원본은 HIS-06.
- **수용기준**: 5회 연속 캡쳐 시 스트립에 **트리거 순서대로** 5개가 좌→우로 쌓이고 5번째가 보인다. 순번 누락·역전 없음.
- **구현 힌트**: `ObservableCollection<T>` + `Dispatcher.Invoke`(또는 `BindingOperations.EnableCollectionSynchronization`).

#### HIS-02 · 항목 재선택 / 미리보기 (Must)
- **설명**: 썸네일 클릭 시 "현재 선택"으로 만들고 메인 미리보기에 **EffectiveImage**를 표시.
- **입력**: 클릭 / 좌우 화살표. **출력**: `SelectedItem` 갱신, 미리보기 렌더.
- **단축키**: `←`/`→`(이전/다음), `Home`/`End`(처음/마지막).
- **수용기준**: 클릭한 썸네일이 하이라이트되고 미리보기가 즉시 바뀐다.

#### HIS-03 · 편집기로 열기 (재편집) (Must)
- **설명**: 선택 항목을 주석 편집기(§3)로 전달해 연다. 더블클릭/"편집" 버튼.
- **동작 흐름**: 더블클릭 → `EditorWindow` 오픈(CaptureItem 전달) → 편집 후 저장 시 editor가 **EditedComposite(Frozen) 채움 + IsEdited=true 마킹 + 썸네일 재생성을 HistoryService에 통지**.
- **입력**: 더블클릭 / `Enter`. **출력**: 편집기 창, 갱신된 썸네일(EditedComposite 기준).
- **예외/엣지**: 이미 편집 중인 항목 재오픈 시 기존 창 포커스(중복 창 방지). **레이어는 비영속**(닫으면 드로잉 객체 폐기 → 개별 주석 재편집 불가, v1 한계).
- **수용기준**: 더블클릭으로 편집기가 열리고, 편집·저장 후 스트립 썸네일이 편집 결과로 갱신된다.

#### HIS-04 · 삭제 / 전체 비우기 (Must)
- **설명**: 단일/다중 선택 삭제, 전체 비우기.
- **입력**: `Delete`, 컨텍스트 메뉴. **출력**: 컬렉션 제거 + 연결 임시파일(Original/EditedComposite) 정리.
- **예외/엣지**: ① 삭제 항목이 편집기에 열려 있으면 경고 후 닫기. ② "전체 비우기"는 확인 다이얼로그(되돌릴 수 없음). ③ `TempFilePath` 파일 동반 삭제(누수 방지).
- **수용기준**: 삭제 후 스트립·메모리·임시폴더에서 사라진다. 전체 비우기 후 컬렉션이 비고 임시폴더 정리.

#### HIS-05 · 세션 영속화 (재시작 복원) (Should)
- **설명**: 재시작해도 직전 세션 히스토리(순서·편집 결과 포함)를 복원.
- **영속 정책(2층위)**: **(레이어) 비영속** — 드로잉 객체는 닫으면 폐기, 개별 주석 재편집 불가(v1 한계, v2에서 레이어 영속). / **(합성본) 영속** — `EditedComposite`(평탄화 PNG)는 영속 폴더 저장. → 한 줄 요약 **"편집 결과는 보이지만 더 못 고친다"**.
- **동작 흐름**: 캡쳐/편집 시 Original·EditedComposite를 `%LOCALAPPDATA%\SnapStack\history\{guid}*.png`에 저장 + 인덱스(JSON: Id·Sequence·CapturedAt·IsEdited·Original경로·EditedComposite경로) 기록. 시작 시 인덱스를 **Sequence 순서로** 읽어 썸네일 lazy 복원, IsEdited 항목은 **EffectiveImage(=EditedComposite)** 로 표시.
- **예외/엣지**: ① 인덱스 손상/파일 누락 → 해당 항목 스킵, 나머지 복원. ② 보존 개수 상한(설정, 예: 최근 50건) 초과분 오래된 것부터 정리. ③ 영속화 OFF 시 종료 즉시 임시폴더 비움.
- **수용기준**: 캡쳐 3건(1건 편집) 후 재시작 시 동일 순서로 3건 복원, 편집본 그림 유지(재편집은 불가).
- **구현 힌트**: HIS-06 메모리절약 임시파일을 영속 폴더로 승격 + 인덱스 1줄이면 됨(추가 비용 미미). 미구현 시 '원본만 영속/편집본 휘발'로 degrade해도 EffectiveImage 폴백으로 일관.

#### HIS-06 · 메모리 관리 (Should)
- **설명**: 누적에 따른 메모리 증가 억제. **메모리엔 썸네일만 상주**, 원본·편집본은 임시파일 두고 **lazy load**.
- **동작 흐름**: Add 시 ① 썸네일(긴 변 256px, **Frozen**) 상주, ② Original(및 IsEdited면 EditedComposite)을 `TempFilePath`(PNG)로 비동기 저장 후 메모리 참조 해제, ③ 선택/편집/복사 시 `TempFilePath`에서 **EffectiveImage** 디코드. 편집기는 합성본을 HistoryService에 넘기는 즉시 자기 임시 비트맵 해제(소유권 인수).
- **수용기준**: 30건 캡쳐 후에도 작업 집합이 (썸네일×30 + 현재 선택 EffectiveImage 1건) 수준 유지.
- **예외/엣지**: lazy load 시 임시파일 소실(외부 삭제) → 썸네일만 표시 + "원본 없음" 표식, 복사/저장 비활성.
- **구현 힌트**: 썸네일 디코드 `BitmapDecoder`+`DecodePixelWidth`; 모든 공유 비트맵 `Freeze()`.

### 4.2 클립보드 복사 (필수 기능 #3) — `CLP`

> 편집/캡쳐 이미지를 클립보드에 **이미지로** 담는다. 외부 앱 붙여넣기(#4)의 성패는 **다중 포맷 등록**에 달려 있다(§4.3 매트릭스).

#### CLP-01 · 이미지 클립보드 복사 (다중 포맷) (Must) ★핵심
- **설명**: 이미지를 **하나의 `DataObject`에 여러 포맷으로 동시 등록**해 클립보드에 올린다. 등록 포맷: **PNG(투명 보존) + CF_DIBV5(알파 보존 DIB) + CF_DIB/Bitmap(광범위 호환·흰색 매트 폴백)**.
- **진입점(2종)**:
  1. **편집기 경유** — 편집기 `Ctrl+C`: `EditorViewModel.FlattenToBitmap()`(Pbgra32 frozen) → `ClipboardService.CopyImage`.
  2. **오버레이 즉시복사** — 캡쳐 오버레이 `Ctrl+Enter`(전역 아님·오버레이 내부): 캡쳐 확정 직후 편집 안 한 원본(IsEdited=false)을 그대로 → `ClipboardService.CopyImage`. **실행 순서: 히스토리 Add 선행 → 그다음 ClipboardService 호출.**
  - 히스토리에서 직접 복사(편집기 미경유)도 **EffectiveImage** 인코딩.
- **동작 흐름**: ① 입력 비트맵 확보(EffectiveImage 또는 평탄화본) → ② `var data = new DataObject();` → ③ **PNG**: `PngBitmapEncoder` → `MemoryStream` → `data.SetData("PNG", stream)`(최우선, 알파 무손실, 최신 Office/그림판 선호) → ④ **CF_DIBV5**: `BITMAPV5HEADER` 패킹 DIB(알파 advertise, premultiplied) 등록 → ⑤ **CF_DIB/Bitmap**: 호환용(흰색 매트) 등록 → ⑥ `Clipboard.SetDataObject(data, copy:true)`.
- **입력**: `BitmapSource`. **출력**: 시스템 클립보드(다중 포맷). **반환**: `bool`(성공/실패) → 호출 측이 토스트.
- **단축키**: **`Ctrl+C`**(편집기/미리보기), **`Ctrl+Enter`**(오버레이 즉시복사).
- **예외/엣지**:
  - **블루 배경 아티팩트**: 단순 `SetImage`는 투명부가 파란색/검정으로 깨짐 → **PNG+DIBV5 우선 등록**으로 회피, DIB 폴백은 **의도적 흰색 매트**.
  - **premultiplied alpha**: Chrome/Firefox 등은 DIBV5를 premultiplied로 기록(사실상 표준). `Pbgra32` 입력이면 정합.
  - **v1 알파 단순화**: 캡쳐 원본은 BitBlt 불투명 BGRA이고 **v1 자르기는 직사각형만**이라 편집본 최종 알파도 사실상 불투명(255). 따라서 v1은 DIB만으로도 색·투명 깨짐 없이 안전하며, **PNG/DIBV5 알파 경로는 향후 비사각 크롭/투명 배경 도입 대비 인프라**로 유지(이미 Pbgra32 보유, 추가 비용 미미).
  - 클립보드 잠금(타 앱 점유) → 짧은 백오프 재시도 후 실패 시 토스트.
  - 대형 이미지 → PNG 인코딩 비동기(입력이 Frozen이라 백그라운드 안전), UI 블로킹 방지.
- **수용기준**: 편집본(또는 원본) 복사 → §4.3의 그림판/한글/Word/카톡/Discord에 붙여넣어 **이미지 표시**. (향후 투명 도입 시 PNG 수용 앱에서 투명 보존.)
- **구현 힌트**: `DataObject.SetData(format, data)` 포맷별 등록; 표준 포맷명 "PNG" 사용해 소비 측이 PNG를 먼저 보게.

#### CLP-02 · 클립보드에서 이미지 가져오기 (Could)
- **설명**: 외부에서 복사한 이미지를 히스토리/편집기로 **받기**(역방향). 메인/편집기에서 `Ctrl+V`.
- **동작 흐름**: `Clipboard.GetDataObject()` → **PNG 우선**(투명 보존) → 없으면 DIBV5 → CF_DIB → `GetImage()` 폴백 → 새 CaptureItem으로 적재.
- **단축키**: `Ctrl+V`. **수용기준**: 그림판/브라우저 복사 이미지를 `Ctrl+V`로 히스토리에 추가.
- **비고**: 입력은 부가(Could). 출력(CLP-01)이 필수 축.
- **구현 힌트**: PNG부터 검사 — .NET Bitmap/DIB 경로는 알파 못 살리므로 raw PNG 스트림 우선 디코드.

### 4.3 외부 앱 붙여넣기 (필수 기능 #4) — `PST`

> SnapStack은 붙여넣는 주체가 아니라 **클립보드를 통해 외부 앱이 받게** 한다. #4의 본질은 **"어떤 포맷을 어떤 앱이 받는가"의 호환성 보장**이며, 이는 CLP-01의 다중 포맷 등록 품질에 달려 있다.

#### PST-01 · 외부 앱 Ctrl+V 호환 (Must) ★핵심
- **설명**: CLP-01로 등록한 다중 포맷을 대상 앱이 지원 포맷으로 골라 받는다. SnapStack은 "복사만 잘 해두면" 외부 `Ctrl+V`가 성립.
- **동작 흐름**: (SnapStack) CLP-01 복사 → (사용자) 대상 앱 `Ctrl+V` → 대상 앱이 우선 포맷(보통 PNG > DIBV5 > DIB > Bitmap) 선택 → 표시.

##### 붙여넣기 호환성 검증 매트릭스 (대상앱 × 지원포맷 × 비고)

| 대상 앱 | PNG(투명) | CF_DIBV5(알파) | CF_DIB/Bitmap | 기대 결과 | 비고 / 리스크 |
|---------|:---:|:---:|:---:|------|------|
| **그림판(Win11)** | ✅ 선호 | △ | ✅ | 표시·(투명 보존) | 구버전 Paint는 PNG 미지원 → DIB 폴백 |
| **그림판 3D / Paint.NET** | ✅ | △ | ✅ | 표시 | Paint.NET·Word2010류는 DIB만 받으면 투명 손실 → PNG 필요 |
| **한글(HWP)** | ✅ | △ | ✅ | 표시 | DIB로도 삽입, 투명 보존은 PNG 경유가 안전 |
| **MS Word** | ✅ 선호 | △ | ✅ | 표시·(투명 보존) | 최신 Office는 PNG 클립보드 선호 |
| **카카오톡(채팅창)** | ✅ | ? | ✅ | 이미지 전송 미리보기 | 버전별 차이 큼 → PNG+DIB 둘 다 필요 |
| **Discord(입력)** | ✅ | ? | △ | 첨부 미리보기 | PNG/비트맵 수용, DIB 단독 불안정 |
| **메모장(Notepad)** | ❌ | ❌ | ❌ | **붙여넣기 불가(정상)** | 텍스트 전용 — 버그 아님 |
| **브라우저 입력(Gmail 등)** | ✅ | ? | △ | 인라인 삽입 | PNG 의존도 높음 |

> 범례: ✅ 받음/선호, △ 받지만 알파 손실 가능, ? 버전별 불확실, ❌ 미지원.
> **각주(v1)**: v1은 캡쳐 원본·편집본 모두 불투명이라 **모든 앱이 DIB로도 안전**. '투명 보존' 컬럼은 향후 투명 도입 시 발휘되는 PNG/DIBV5 경로 기준이며, v1 인프라로 미리 깔아둔다.
> **결론**: **PNG 1순위 + DIBV5 + DIB(흰색 매트) 3중 등록**으로 구·신 앱을 모두 커버.

- **예외/엣지**: ① 메모장 등 텍스트 전용 앱은 "불가가 정상". ② 일부 메신저는 붙여넣기 대신 "파일 첨부"만 → §SAV 저장 후 드래그 안내. ③ 클립보드 차단 RDP 등 한계 명시.
- **수용기준**: CLP-01 복사본이 **그림판·한글·Word·카톡·Discord 5개 앱에 붙여넣기 성공**. 메모장 미지원이 정상 동작으로 확인.

##### 호환성 리스크와 대응 (강조)
- **리스크 R1(DEVELOPMENT_PLAN §9)**: 일부 외부 앱 붙여넣기 실패/투명 깨짐 → **영향 높음(필수 기능)**.
- **대응(확정)**: ① **PNG + CF_DIBV5 + CF_DIB(흰색 매트) 3중 등록**. ② DIB 폴백 의도적 흰색 합성으로 블루/블랙 배경 차단. ③ **대표 앱 매트릭스를 회귀 체크리스트화**(§10·§11 연계). ④ premultiplied alpha 정합(`Pbgra32`→DIBV5).

### 4.4 파일 저장 / 내보내기 — `SAV`

> 클립보드 경유가 안 되는 워크플로(첨부 업로드·보관)용. 평탄화 계약(§4.0)에 따라 **EffectiveImage(합성된 비트맵)** 를 인코딩.

#### SAV-01 · 빠른 저장 (Quick Save) (Must)
- **설명**: 현재 이미지를 **기본 폴더 + 파일명 규칙**으로 다이얼로그 없이 즉시 저장.
- **동작 흐름**: `Ctrl+S` → 기본 포맷(기본 PNG)·기본 폴더·자동 파일명으로 `FileExportService.Save` → 호출 측이 토스트(경로 표시).
- **입력**: EffectiveImage. **출력**: 파일 1개. **단축키**: **`Ctrl+S`** / 오버레이 `Shift+Enter`.
- **수용기준**: `Ctrl+S` 한 번으로 기본 폴더에 `SnapStack_yyyyMMdd_HHmmss_NNN.png` 생성.

#### SAV-02 · 다른 이름으로 저장 (Save As) (Must)
- **설명**: `SaveFileDialog`로 경로·파일명·포맷(**PNG / JPG / BMP**) 선택 저장.
- **동작 흐름**: `Ctrl+Shift+S` → 다이얼로그 → 선택 인코더로 저장.
- **포맷별 인코더**: PNG=`PngBitmapEncoder`(무손실·투명), JPG=`JpegBitmapEncoder`(품질 슬라이더, **투명→흰색 합성**), BMP=`BmpBitmapEncoder`(무손실).
- **단축키**: **`Ctrl+Shift+S`**.
- **예외/엣지**: ① JPG/BMP 알파 미지원 → 저장 전 **매트 색(기본 흰색) 합성** 후 인코딩, 고지. ② 동일 파일명 → 덮어쓰기 확인 또는 `(2)` 접미. ③ 권한/디스크 부족 → 오류 토스트.
- **수용기준**: PNG 저장 시 (투명 보존), JPG/BMP 저장 시 흰색 배경 평탄화되어 정상 열림.

#### SAV-03 · 파일명 규칙 / 기본 폴더 (Must)
- **설명**: 자동 파일명 템플릿·기본 저장 폴더를 설정에서 지정.
- **기본 템플릿**: `SnapStack_{yyyyMMdd}_{HHmmss}_{seq:000}`. 토큰: `{yyyy}{MM}{dd}{HH}{mm}{ss}{seq}{app?}`.
- **기본 폴더**: `%USERPROFILE%\Pictures\SnapStack\`(없으면 생성). 설정 변경 가능.
- **예외/엣지**: 금지 문자(`\ / : * ? " < > |`) 자동 치환; seq는 같은 초 충돌 방지.
- **수용기준**: 같은 초 2회 저장에도 파일명 미충돌(seq 증가).

#### SAV-04 · 자동 저장 (Auto Save) (Could)
- **설명**: 캡쳐/편집 완료마다 기본 폴더에 자동 저장(설정 ON 시, 기본 OFF).
- **예외/엣지**: 대량 캡쳐 시 디스크 폭증 → 보존 개수/용량 상한. §HIS-05 영속화 파일과 **중복 저장 회피**(경로 재사용).
- **수용기준**: 자동저장 ON에서 캡쳐 시 기본 폴더에 파일 자동 생성.

## 5. 시스템 통합·설정 — `SYS`

> 전역 단축키·트레이 상주·설정 영속. 캡쳐 도구는 평소 트레이에 머물다 전역 단축키로 즉시 동작한다.

#### SYS-01 · 전역 단축키 (Must)
- **설명**: 앱이 포커스 없어도 동작하는 전역 핫키 등록/해제, 충돌 처리, 사용자 커스터마이즈.
- **동작/엣지**: Win32 `RegisterHotKey` + `HwndSource.AddHook`로 `WM_HOTKEY` 수신. 등록 실패(타 앱 선점) 감지 → 설정에서 재지정 안내·해당 키만 비활성. 앱 종료 시 `UnregisterHotKey`.
- **기본값**: §2.3 캡쳐 단축키(Ctrl+Shift+A/W/F/G/R/D). **전역키 철학 = B안(Ctrl+Shift+\* 통일)** — PrtScn OS 충돌 회피.
- **수용기준**: 다른 앱이 떠 있어도 전역 단축키로 캡쳐가 시작된다. 충돌 시 명확히 안내.

#### SYS-02 · 시스템 트레이 상주 (Must)
- **설명**: 트레이 아이콘 상주, 우클릭 메뉴(영역/전체/창/지연 캡쳐·설정·열기·종료), 더블클릭 동작(기본 영역 캡쳐), 캡쳐 완료 풍선/토스트.
- **동작/엣지**: 창 닫기 = 트레이로 최소화(완전 종료는 메뉴). 단일 인스턴스 보장(중복 실행 시 기존 인스턴스 활성화).
- **수용기준**: 트레이 메뉴에서 모든 캡쳐 모드·설정·종료가 동작한다.
- **구현 힌트**: `H.NotifyIcon.Wpf`(`TaskbarIcon`), `Mutex` 단일 인스턴스.

#### SYS-03 · 시작 프로그램 자동 등록 (Should)
- **설명**: Windows 시작 시 자동 실행(설정 토글).
- **동작/엣지**: 레지스트리 `HKCU\...\Run`에 등록/해제. 설정 OFF 시 제거.
- **수용기준**: 토글 ON 후 재부팅 시 트레이에 자동 상주.

#### SYS-04 · 설정 화면 (Must)
- **설명**: 탭 구성 — **일반**(시작시 실행·창 닫기 동작·언어), **단축키**(전 기능 재지정·충돌 검사), **저장**(기본 폴더·포맷·파일명 템플릿·자동저장), **캡쳐**(지연 시간·캡쳐 후 동작=편집기 자동열기/미리보기/즉시복사·확대경), **정보**(버전·라이선스).
- **수용기준**: 각 설정 변경이 즉시 반영·영속된다.

#### SYS-05 · 설정 영속화 (Must)
- **설명**: 설정을 JSON으로 저장/로드.
- **동작/엣지**: `%APPDATA%\SnapStack\settings.json`. 손상 시 기본값으로 복구(백업 1세대). 스키마 버전 필드로 마이그레이션.
- **수용기준**: 재시작 후 모든 설정이 유지된다.
